## Code of Conduct

Be respectful, constructive, and collaborative in all discussions and reviews.

## Contributing Workflow

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make your changes in focused commits.
4. Validate changes locally.
5. Open a pull request with a clear summary.

## Development Setup

1. Install dependencies:

```bash
npm install
```

2. Run the app:

```bash
npm start
```

## Coding Guidelines

- Keep changes scoped to one purpose per pull request.
- Preserve existing coding style and conventions.
- Avoid unrelated refactors in feature/fix PRs.
- Add concise comments only when logic is non-obvious.

## Issue Reports

When opening an issue, include:

- Expected behavior
- Actual behavior
- Steps to reproduce
- OS and Node.js version
- Screenshots or logs when applicable

## Pull Request Checklist

- [ ] Changes are tested locally
- [ ] No unrelated files are modified
- [ ] UI/behavior changes are described clearly
- [ ] Any new behavior is documented in [README.md](README.md)
